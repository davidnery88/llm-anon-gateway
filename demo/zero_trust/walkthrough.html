Démo zero-trust — pas à pas

Setup en haut (4 étapes), puis 4 scènes de démo. Les badges Terminal = à taper dans bash ; Claude Code = à coller dans une session claude.

Setup — à faire une fois
1

Lancer le gateway Terminal

Démarre gateway (port 8001) + postgres.

cd ~/Documents/llm-anon-gateway && docker compose up -d gateway postgres
Attendu : deux containers Started. Patiente ~15 sec, puis vérifie : 
curl -sS -o /dev/null -w "HTTP %{http_code}\n" http://127.0.0.1:8001/docs
Attendu : HTTP 200.
2

Lancer le sidecar Terminal

Démarre le sidecar local (NER + Redis ephémère).

⚠ Première fois : build de l'image ~5-10 min. Les fois suivantes c'est instantané.
cd ~/Documents/llm-anon-gateway && bash sidecar/install.sh --gateway http://host.docker.internal:8001
Attendu : ligne finale [OK] Installation terminée.
3

Reseed la base SQLite de démo Terminal

S'assure que la DB demo/demo.sqlite est à jour (25 clients, 15 sinistres, commentaires libres avec PII clairsemées).

cd ~/Documents/llm-anon-gateway && python3 demo/seed_db.py
Attendu : DB créée : .../demo.sqlite avec les comptes de chaque table.
4

Lancer Claude Code via le proxy Terminal

La variable ANTHROPIC_BASE_URL redirige Claude Code vers le proxy local au lieu d'api.anthropic.com. Tout le trafic — prompts utilisateur, system prompt, sorties de Bash/Read, résultats MCP, etc. — est intercepté par le sidecar, anonymisé en sortie, et désanonymisé en streaming au retour pour ton affichage.

cd ~/Documents/llm-anon-gateway && ANTHROPIC_BASE_URL=http://127.0.0.1:8787 claude
Attendu : Claude Code démarre normalement. Le fichier .claude/settings.local.json du projet active aussi le MCP anon-gateway (outil query_db pour l'analyse de DB). Tape /mcp dans Claude pour vérifier que anon-gateway est connecté.
⚠ Important : ANTHROPIC_BASE_URL est défini à la volée pour cette session uniquement. Si tu ouvres un autre terminal sans cette variable, Claude Code parle direct à Anthropic et il n'y a aucune anonymisation. C'est volontaire — l'anonymisation est strictement opt-in à ce projet.

Le scénario : un collaborateur copie-colle un extrait CSV depuis Excel et demande une analyse. Tu vas voir que le tableau est anonymisé avant d'arriver chez Anthropic, mais que tu reçois la réponse en clair dans ton terminal.

Colle ce prompt entier (avec le tableau) dans Claude Code :

J'ai exporté ce tableau de clients depuis Excel, dis-moi qui a le plus gros risque de résiliation et pourquoi :

docker logs --tail=20 anon-sidecar | grep -i anonymize
2

Phrase libre avec PII Claude Code

Variante plus rapide : un simple prompt conversationnel.

Peux-tu me rédiger un email professionnel à Julien Maillard (julien.maillard@bluewin.ch) qui habite Chemin des Lilas 8, 1003 Lausanne, pour lui confirmer le bon traitement de son dossier d'assistance suite à l'incident sur la route entre Echallens et Yverdon ? Mentionne aussi que sa compagne Sophie Renaud a été informée.
Attendu : l'email rédigé par Claude sort vers Anthropic avec des placeholders, mais le proxy te ré-affiche localement la version avec les vrais noms — prête à copier-coller dans Outlook.
3

Analyse de la base de données Claude Code

Maintenant le test important : Claude analyse une DB SQLite réelle. Il doit utiliser le MCP query_db (et pas le Bash sqlite3 natif) — c'est query_db qui passe par le sidecar et anonymise les résultats avant de les rendre visibles à Claude.

Utilise l'outil MCP query_db (anon-gateway) pour analyser la base de démo. Donne-moi :
1. Le top 5 des clients par nombre de sinistres (jointure clients + contrats_assurance + sinistres)
2. Pour chacun, le détail du dernier sinistre (description, montant)
3. Une analyse des commentaires libres dans la table clients : quels patterns voient les agents (médecins fréquents, types de contacts d'urgence, etc.) ?

NE fais PAS de sqlite3 via Bash — utilise exclusivement le tool MCP query_db pour garantir que les PII restent locales.
Ce que tu vois : Claude appelle query_db plusieurs fois. À chaque appel, le résultat retourné contient un CSV où tous les noms, emails, téléphones, IBAN, etc. sont déjà remplacés par [PERSONNE_N], [EMAIL_N], etc. Claude analyse ces données pseudonymisées et te retourne un rapport. Le proxy décode tout en streaming pour ton terminal — tu vois les vrais noms en local, Anthropic n'a vu que les placeholders.
Vérification : dans un autre terminal, regarde combien d'appels /anonymize le sidecar reçoit pendant cette scène — chaque query_db en génère un.
4

Construction d'un pipeline de données Claude Code

Le test final : Claude doit produire du code Python qui sera exécuté en production. Il a besoin d'explorer la DB pour comprendre la structure, voir des exemples, et écrire un script qui tient debout.

Construis-moi un pipeline Python qui :

1. Lit la base de démo (utilise query_db du MCP anon-gateway pour comprendre le schéma et voir des exemples de données — interdit d'utiliser sqlite3 en direct)
2. Pour chaque sinistre clos en 2025, calcule le délai entre date_sinistre et résolution (suppose résolution = date la plus récente d'intervention liée au même client)
3. Détecte les sinistres « suspects » : montant > 10000 CHF ET description contenant un nom de tiers (autre personne mentionnée que le client lui-même)
4. Génère un rapport CSV final avec : no_sinistre, montant, délai_jours, tiers_mentionnés, score_suspicion

Le pipeline doit être PRÊT À TOURNER EN PROD : pas de PII en clair dans les logs, pas de noms en dur dans le code, et doit pouvoir traiter 100k lignes sans charger toute la DB en mémoire.
Ce que tu vois : Claude alterne entre appels query_db exploratoires (toujours anonymisés) et écriture de code. Le code final qu'il produit ne contient aucun nom de client en dur, juste du code générique qui consomme les données pseudonymisées.
🛡

Bonus — démontrer le fail-safe Terminal

Pour prouver que rien ne fuit en cas de panne du sidecar : on l'arrête brutalement et on retape un prompt.

docker stop anon-sidecar

Puis dans Claude Code, retape n'importe quel prompt. Attendu : Claude Code affiche une erreur 503 Anonymization failed — request blocked (fail-safe). Le proxy refuse de forward la requête vers Anthropic si l'anonymisation a crashé. Pour redémarrer :

docker start anon-sidecar
⚙

En cas de souci Terminal

Logs sidecar :

docker logs -f anon-sidecar

Tester directement le sidecar (smoke) :

curl -sS http://127.0.0.1:8787/anonymize \
  -H "X-Sidecar-Token: $(cat ~/.config/anon-sidecar/token)" \
  -H "Content-Type: application/json" \
  -d '{"text":"Julien Maillard habite Lausanne."}'

Tout stopper :

cd ~/Documents/llm-anon-gateway && bash sidecar/uninstall.sh && docker compose down

Config démo dans .claude/settings.local.json (MCP anon-gateway) + variable d'env ANTHROPIC_BASE_URL définie à la volée — l'anonymisation est strictement opt-in à ce projet, ne pollue pas la config Claude Code globale.
